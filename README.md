# MALDI-TOF Classifier – Example Data Repository

## Overview

This repository contains example data for the maldi-tof-classifier project.

**License:** MIT

The maldi-tof-classifier project includes a CLI tool as well as a Python-based API
for the classification of MALDI-TOF spectra, as generated by a Shimadzu 8300 mass spectrometer.

The corresponding repository can be found at:
[GitHub maldi-tof-classifier](https://github.com/ofmk94/maldi-tof-classifier)

---

## Repository Structure

### data_train

Contains training data for a classification model.

Each measurement is represented by:

- one **.txt file** containing the full MALDI-TOF spectrum.
- one **.csv file** containing the peaks extracted by the mass spectrometer software.

Thus, for each individual measurement, both a full-spectrum file (`.txt`) and a corresponding peak-data file (`.csv`) are available.

The subdirectories
`data_train/5`, `data_train/100`, `data_train/C`
contain the measurements of the respective classes to be learned:

- class 5 (72 measurements)
- class 100 (72 measurements)
- class C (36 measurements)

---

### data_pred

Contains unlabeled data whose class can be predicted by the trained model.

As in `data_train`, each measurement is represented by:

- one **.txt file** containing the full spectrum
- one **.csv file** containing the peaks extracted by the mass spectrometer software

This directory directly contains the measurements:

- 24 from class 5
- 24 from class 100
- 12 from class C

---

### cli_files

Contains the configuration file and the generated output files of the CLI tool from the maldi-tof-classifier project:

- **config.yaml**
  contains the configuration for the CLI tool, here with some typical example values

- **pipeline.joblib**
  contains the trained pipeline with the classification model and preprocessing steps, as well as a mapping between the labels and the numerically encoded classes

- **predictions.csv**
  contains the predictions of the model for the data in `data_pred` with the two columns:
  - `prediction` (class)
  - `filename` (corresponding file)

- **training_performance.txt**
  contains the achieved performance metrics of the trained classifier:
  - accuracy
  - precision
  - recall
  - F1 score
  - confusion matrix

---

### original_data

Contains all unsorted raw data, including:

- full spectra (`.txt`)
- peak data (`.csv`)
- alternative preprocessing steps
- different data formats
- the assignment of measurements to the corresponding cell cultures (5A, 5B, 100A, 100B, C)

See also the information below regarding the origin of the data.

---

## Data Origin

The spectra contained in the data originate from MALDI-TOF measurements of extracellular vesicles from colorectal cancer cells that were treated with different amounts of the chemotherapeutic agent 5FU.

The experiments for generating the data were conducted under the supervision of Dr. Harald Kühnel at the University of Applied Sciences Campus Vienna.

The measurement data were used by Oliver Klein in the context of his master’s thesis
*"Machine Learning Pipelines for Extracellular Vesicle Fingerprint Classification"*
in the Bioinformatics master’s program at the University of Applied Sciences Campus Vienna.

---

## Experimental Setup

- Five cultures of cells of the colorectal cancer cell line CACO2 were cultivated.

- Two cultures were treated with a protocol of 5 µM (day 1), then 30 µmol (day 2) of the chemotherapeutic agent 5FU.
  -> class **"5"**

- Two further cultures were treated with 100 µM (day 1), then 200 µmol (day 2) of 5FU.
  -> class **"100"**

- One culture was not treated and represents the control group.
  -> class **"C"**

- Extracellular vesicles were extracted from the cultures by ultracentrifugation.

- Using a Shimadzu 8300 MALDI-TOF mass spectrometer, 48 measurements were performed for each cell culture.

- For each measurement, the output includes:
  - one text file with the full spectrum.
  - one CSV file with the peaks extracted by the mass spectrometer software.
